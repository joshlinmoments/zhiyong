# 智勇人才班教材 — 互動講課網站設計規格

- **日期**：2026-06-18
- **作者**：Josh Lin（林瑋揚）
- **狀態**：設計已核可，待撰寫實作計畫

---

## 1. 目的

把《智勇人才班教材》變成一個可線上閱讀、且**講者可在講課時即時加註**的網站。
讀者（含長輩）可舒適閱讀；講者可針對任意詞語/句子加入敘述、圖片、外部連結，
作為授課輔助；註解存於雲端，所有人即時看到。

## 2. 兩種模式

| 模式 | 對象 | 能力 |
|---|---|---|
| **瀏覽模式**（預設） | 所有人 | 閱讀全書、看到既有註解、調整字級、章節跳轉 |
| **編輯模式** | 輸入講者密碼解鎖 | 在瀏覽功能之上，選取文字新增/編輯/刪除註解 |

## 3. 頁面與流程

### 3.1 登陸頁（封面 + 目錄）
- 全幅中國風封面：水墨長卷／樓閣祥雲淡背景、**智勇印章 logo**、書名「智勇人才班教材」。
- **卷首區**：樞紐四訓、序文〈樞紐慈諭〉兩個入口。
- **章節格**：壹～拾陸共 16 章，點任一章 → 進入該章閱讀頁。
- 常駐：模式切換（瀏覽 ⇄ 編輯）、字級控制（A− A A+）。

### 3.2 閱讀頁
- 左側目錄側欄（章 → 節，可收合、可跳節）；主區呈現章標題、節標題、內文、引用。
- 有註解的文字以**朱紅底/底線**標示；點擊 → 右側面板顯示該註解的敘述、圖片、連結。
- 右上：回封面、模式切換、字級控制。
- 卷首頁（樞紐四訓／序文）為特殊版型，無節側欄。

## 4. 視覺系統（中國風）

### 色板
| 角色 | 色 | Hex |
|---|---|---|
| 主色 | 青碧 | `#6E9C8A` |
| 背景 | 宣紙米黃 | `#F4ECDA` |
| 點綴 | 赭黃 | `#C8A063` |
| 強調/印章 | 朱印紅 | `#B23A2E` |
| 文字 | 墨黑 | `#2B2724` |

### 字體
- **標題**：書法/楷書感字體（可用 Google Fonts 的 `Noto Serif TC` 粗體或楷體 webfont），搭配朱紅印章與框線。
- **內文**：宋體/明體（`Noto Serif TC`），寬鬆行距，求易讀。
- 詩偈/聖訓延用「印章 + 直書標題 + 框線」古籍感引用樣式。

### 素材
- 品牌標誌：`Resources/智勇2.png`（智勇書法 + 朱紅印章）。
- 背景底紋：自 `Resources/` 既有中國風圖片擇一處理為低透明度底紋（部署時打包進站內）。

## 5. 字級控制（長輩友善）
- 常駐 **`A− A A+`**，4 級：標準 / 大 / 特大 / 超大。
- 即時縮放整頁閱讀文字（透過 CSS 變數 `--reading-scale`）。
- 設定記憶於 `localStorage`，下次自動套用。
- 按鈕大、對比高，行動裝置可單手操作。

## 6. 註解模型

### 6.1 錨定方式
- 註解錨定在「某內容單元（rowId）中的字元範圍」。
- 因正文為固定的靜態 JSON，字元位移（startOffset/endOffset）穩定可靠。
- 同時保存 `selectedText` 供顯示與校驗（位移失準時以文字比對回退）。

### 6.2 內容資料（content.json，隨站打包）
```json
{
  "id": "ch01-sec01-003",
  "chapter": "壹、人才的自我覺悟",
  "section": "一、前言",
  "level": "內文",
  "text": "人才是道場的棟樑幹部。"
}
```
- 卷首（樞紐四訓 / 序文）以特殊 `level` 標記。
- 由先前製作的 Excel（章/節/階層/內容）程式化轉出。

### 6.3 註解資料（Realtime Database，路徑 `annotations/{pushId}`）
```json
{
  "rowId": "ch01-sec01-003",
  "selectedText": "棟樑",
  "startOffset": 5,
  "endOffset": 7,
  "note": "（敘述文字，支援換行/簡單格式）",
  "images": [{ "url": "https://…", "type": "upload|link" }],
  "links":  [{ "label": "延伸閱讀", "url": "https://…" }],
  "author": "Josh",
  "createdAt": 1718700000000,
  "updatedAt": 1718700000000
}
```
- 以 `push()` 產生唯一 key；前端對 `annotations` 掛 `onValue` 即時監聽。
- 顯示時依 `rowId` 分組，套用到對應內容單元的字元範圍。

## 7. 後端與安全（Firebase）

- **新開一個獨立的 Firebase 專案**（與既有 `tao-why` 分離；專案名建議 `zhiyong-jiaocai` 之類）。
  - 由 Josh 在 Firebase 主控台建立；Claude 提供逐步指引，Josh 把設定值（apiKey、databaseURL、storageBucket…）貼回。
- **Realtime Database**：存註解，前端掛 `onValue` 即時監聽；任何人新增/修改，所有瀏覽者即時更新。
  - 與既有 `tao-why`／`cct-translator` 相同、已驗證可行的模式。
- **Firebase Storage**：存講者上傳的圖片（另支援直接貼圖片網址）。
- **簡易密碼鎖（防君子）**：
  - 編輯模式由前端密碼解鎖；資料庫採開放寫入規則（`.read`/`.write` 開放）。
  - 密碼僅為 UI 閘門，**非真正權限控管**——適合小型可信講者團隊使用，明確接受此取捨。
  - 解鎖狀態記憶於 `sessionStorage`（關閉分頁即需重新輸入）。
- Firebase 設定值（apiKey 等）屬前端公開資訊，可安全放入靜態站。

## 8. 技術堆疊與部署

- **純靜態前端**：HTML + CSS + 原生 JS（ES modules），**無編譯步驟**。
- Firebase 透過 CDN 的 **modular SDK** 引入。
- **部署**：新 GitHub repo（建議 `zhiyong-web`），GitHub Pages 上線（與 taoteching-web 相同模式）。
- 提供逐步設定指引：建立 Firebase 專案、開 Firestore/Storage/Auth、建立講者帳號、貼上設定值、設定安全規則。
